Scenario-Based Kotlin Coroutines Program Questions

Scenario 1: Network Call on Background Thread

You are building an Android app. You need to fetch user data from a network API.

Task:

  • Write a suspend fun that simulates a network call using delay()
  • Call it from a coroutine running on Dispatchers.IO
  • Print the result on the Main thread

Scenario 2: Parallel API Calls

An app needs to fetch user profile and user posts at the same time.

Task:

  • Create two suspend functions
  • Run them in parallel using async
  • Wait for both results and print them together

Scenario 3: Cancel Long Running Task

A file download is running, but the user leaves the screen.

Task:

  • Launch a coroutine that runs a loop with delay()
  • Cancel the coroutine after 2 seconds
  • Handle cancellation properly

Scenario 4: Exception Handling

A coroutine throws an exception during execution.

Task:

  • Create a coroutine using launch
  • Throw an exception inside it
  • Handle it using CoroutineExceptionHandler

Scenario 5: Structured Concurrency

You start multiple child coroutines inside a parent coroutine.

Task:

  • Launch two child coroutines
  • Cancel the parent coroutine
  • Observe what happens to child coroutines

Scenario 6: Switching Threads

You are doing heavy computation and then updating UI.

Task:

  • Start computation on Dispatchers.Default
  • Switch to Dispatchers.Main using withContext

Scenario 7: Timeout Handling

A network call should not run longer than 3 seconds.

Task:

  • Use withTimeout or withTimeoutOrNull
  • Handle timeout gracefully

Scenario 8: Flow vs Suspend

You need to emit multiple values over time.

Task:

  • Create a Flow that emits numbers every second
  • Collect it inside a coroutine

Scenario 9: Lifecycle-Aware Coroutine (Android)

You want a coroutine to stop automatically when the screen is destroyed.

Task:

  • Use lifecycleScope or viewModelScope
  • Explain why it prevents memory leaks

Scenario 10: Compare Thread vs Coroutine

You need to run 10,000 background tasks.

Task:

  • Explain why coroutines are better than threads
  • Write a small example using repeat() and launch
