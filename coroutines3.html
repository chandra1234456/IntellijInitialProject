Kotlin Coroutines – Coding Assignments

Each assignment includes:

  • Problem Statement
  • Coding Task
  • Interview Expected Explanation

Assignment 1: Network Call on Background Thread

Problem Statement

You are developing an application that fetches data from a remote server. The network call must not block the main thread.

Coding Task

  • Create a suspend fun that simulates a network call
  • Execute it on Dispatchers.IO
  • Print the result

Interview Expected Explanation

In interviews, the key expectation is understanding that:

  • Network operations are IO-bound
  • Dispatchers.IO is optimized for blocking IO tasks
  • suspend fun allows non-blocking suspension without freezing threads
Mention that coroutines keep the app responsive while waiting for results.

Assignment 2: Parallel API Calls

Problem Statement

An application must load user profile and user posts at the same time to reduce waiting time.

Coding Task

  • Create two suspend functions
  • Run them in parallel using async
  • Combine the results

Interview Expected Explanation

Interviewers expect you to explain:

  • async is used for concurrent tasks that return values
  • await() suspends without blocking the thread
  • Parallel execution improves performance compared to sequential calls

Assignment 3: Cancelling a Long-Running Task

Problem Statement

A file download should stop if the user leaves the screen.

Coding Task

  • Launch a coroutine that runs repeatedly
  • Cancel it after a delay

Interview Expected Explanation

The interviewer wants to hear:

  • Coroutines support cooperative cancellation
  • delay() is cancellable
  • Cancellation avoids wasted resources and memory leaks

Assignment 4: Exception Handling in Coroutines

Problem Statement

A coroutine throws an exception during execution.

Coding Task

  • Launch a coroutine
  • Throw an exception
  • Handle it properly

Interview Expected Explanation

Key interview points:

  • launch exceptions are handled immediately
  • CoroutineExceptionHandler handles uncaught exceptions
  • async exceptions are caught on await()

Assignment 5: Structured Concurrency

Problem Statement

Multiple background tasks are started under a parent task.

Coding Task

  • Launch a parent coroutine
  • Launch child coroutines inside it
  • Cancel the parent

Interview Expected Explanation

Interviewers expect:

  • Understanding of parent–child relationship
  • When parent is cancelled, children are cancelled automatically
  • This prevents orphan background tasks

Assignment 6: Switching Threads

Problem Statement

Heavy computation should not run on the main thread.

Coding Task

  • Run computation on Dispatchers.Default
  • Switch context using withContext

Interview Expected Explanation

Expected explanation:

  • Dispatchers.Default is optimized for CPU-intensive work
  • withContext switches threads safely
  • No new coroutine is launched, context is just changed

Assignment 7: Timeout Handling

Problem Statement

A network request should fail if it takes too long.

Coding Task

  • Use withTimeout or withTimeoutOrNull

Interview Expected Explanation

Interviewers look for:

  • Timeout avoids infinite waiting
  • withTimeout throws exception
  • withTimeoutOrNull fails safely

Assignment 8: Flow for Multiple Values

Problem Statement

You need to emit values continuously over time.

Coding Task

  • Create a Flow
  • Emit values periodically
  • Collect them

Interview Expected Explanation

Key explanation:

  • Flow is a cold asynchronous stream
  • Values are emitted only when collected
  • Better than suspend functions for multiple values

Assignment 9: Lifecycle-Aware Coroutines (Android)

Problem Statement

Background work should stop automatically when the UI is destroyed.

Coding Task

  • Use viewModelScope or lifecycleScope

Interview Expected Explanation

Expected answer:

  • Lifecycle-aware scopes cancel coroutines automatically
  • Prevents memory leaks
  • Follows structured concurrency

Assignment 10: Coroutines vs Threads

Problem Statement

You need to run thousands of background tasks efficiently.

Coding Task

  • Launch many coroutines
  • Explain why coroutines are preferred

Interview Expected Explanation

Interviewers expect:

  • Coroutines are lightweight
  • Threads are expensive and limited
  • Coroutines suspend without blocking threads
