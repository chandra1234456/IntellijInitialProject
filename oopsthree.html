📘 1. Basics of OOP in Kotlin

1 What is Object-Oriented Programming (OOP)?🔰

OOP is a programming paradigm based on "objects" containing data (fields) and code (methods). Principles: Encapsulation, Inheritance, Polymorphism, Abstraction.

2 How does Kotlin support OOP principles?

Kotlin fully supports OOP: classes & objects, inheritance (open keyword), interfaces, abstract classes, visibility modifiers, data classes, sealed classes, etc.

3 What is a class in Kotlin?

Blueprint for creating objects. Defines properties and behaviors. Example: class Person(val name: String)

4 What is an object in Kotlin?

An instance of a class. Created using val p = Person("Alice"). Also 'object' keyword for singleton.

5 Difference between class and object in Kotlin.

class is a definition; object is a singleton instance (declaration). Object can be used for static-like behavior or companion.

class A          // just a class
object B { fun f() }  // singleton, B.f()
6 What is a constructor in Kotlin?

Primary constructor declared in class header: class User(val name: String). Secondary constructors using constructor keyword.

7 Primary vs secondary constructor?

Primary: concise, part of header. Secondary: additional constructors, must delegate to primary. Primary can have default values.

class Person(val name: String) {
    constructor(name: String, age: Int) : this(name)
}
8 What is init block?

Initialization block executed after primary constructor. Used for setup logic. Multiple init blocks run in order.

class InitOrderDemo(name: String) {
    init { println("First: $name") }
}
9 Can a Kotlin class exist without a constructor?

Yes, Kotlin generates a default primary constructor with no parameters implicitly.

10 What is a data class? Why used?

Data classes hold data; automatically generate equals(), hashCode(), toString(), copy(). Useful for POJOs/DTOs. Example: data class User(val id: Int, val name: String)

🔒 2. Encapsulation

11 What is encapsulation in OOP?

Bundling data and methods within a class, hiding internal state, exposing only necessary interfaces. Increases security & modularity.

12 How does Kotlin implement encapsulation?

Using visibility modifiers (private, protected, internal, public) and properties with custom getters/setters.

13 Visibility modifiers in Kotlin?

private (class only), protected (class + subclasses), internal (module), public (everywhere, default).

14 Explain private, protected, internal, public examples.
open class Base {
    private val a = 1
    protected val b = 2
    internal val c = 3
    public val d = 4
}
15 How do getters/setters work in Kotlin?

Properties have default getter/setter. Can customize: var name: String = ""; get() = field.uppercase(); set(value) { field = value.trim() }

16 How to make property read-only?

Use val instead of var. Only getter, no setter.

17 Difference between val and var for encapsulation?

val immutable reference (can't reassign). var mutable. Encapsulation: expose read-only with val, mutability via methods if needed.

🌳 3. Inheritance

18 What is inheritance?

Mechanism where one class derives properties and behaviors from another class (superclass). Promotes code reuse.

19 How to achieve inheritance in Kotlin?

Use open for superclass, then class Derived : Base(). All classes final by default.

20 Why are classes final by default in Kotlin?

Effective Java item: favor composition over inheritance. Prevents fragile base class problem. Forces explicit 'open'.

21 What is use of open keyword?

Marks a class or function as inheritable/overridable. Without open, cannot extend or override.

22 How to override a function?

Declare function as open in base, use override keyword in derived class.

open class Parent { open fun greet() = "Hello" }
class Child : Parent() { override fun greet() = "Hi" }
23 What is override keyword?

Mandatory modifier to indicate you are overriding a member, prevents accidental overriding.

24 Can we override a val with a var?

Yes, because var provides getter + setter, val only getter. Overriding val with var is allowed (widening).

open class A { open val x = 5 }
class B : A() { override var x = 10 }
25 What is super keyword?

Refers to superclass implementation. Used in overridden functions: super.method().

26 Can a Kotlin class inherit from multiple classes?

No, Kotlin (like Java) supports single inheritance for classes. Multiple inheritance via interfaces.

🔄 4. Polymorphism

27 What is polymorphism?

Ability of an object to take many forms. Allows methods to behave differently based on actual object type.

28 Types of polymorphism in Kotlin?

Compile-time (method overloading, operator overloading) and Runtime (method overriding).

29 What is method overriding?

Runtime polymorphism: subclass provides specific implementation of superclass method.

30 What is method overloading? Does Kotlin support?

Multiple functions with same name but different parameters in same class. Yes, Kotlin supports overloading.

fun add(a: Int, b: Int) = a+b
fun add(a: Double, b: Double) = a+b
31 How Kotlin achieves runtime polymorphism?

Using inheritance & method overriding. The JVM uses dynamic dispatch to call correct overridden method.

32 Example of polymorphism.
open class Animal { open fun sound() = "Some sound" }
class Dog : Animal() { override fun sound() = "Bark" }
fun makeSound(animal: Animal) = println(animal.sound())
// makeSound(Dog()) prints "Bark"

🎭 5. Abstraction

33 What is abstraction in OOP?

Hiding implementation details and showing only essential features. Achieved via abstract classes & interfaces.

34 What is abstract class in Kotlin?

Class declared with abstract. Cannot be instantiated; may have abstract (no body) and concrete methods.

35 Can abstract class have non-abstract methods?

Yes, abstract classes can have fully implemented methods.

abstract class Base { abstract fun f(); fun g() = "non-abstract" }
36 What is an interface in Kotlin?

Contract defining methods/properties that implementing classes must provide. Can have default implementations (since Kotlin 1.0).

37 Difference: abstract class vs interface?

Abstract class: can hold state (properties), constructor. Interface: no constructor, can have properties but no backing fields (unless override). Multiple interface inheritance allowed.

38 Can interfaces contain method implementations?

Yes, interfaces can have default implementations (concrete methods) and even properties with getters.

39 Can a class implement multiple interfaces?

Yes, Kotlin supports multiple interface implementation: class C : A, B.

40 Resolve method conflicts from multiple interfaces?

Use super<Interface>.method(). Override method and call specific super.

interface A { fun foo() = "A" }
interface B { fun foo() = "B" }
class C : A, B { override fun foo() = super<A>.foo() }

✨ 6. Special Kotlin OOP Concepts

41 What is a sealed class?

Restricted class hierarchies: all subclasses known at compile time. Used for representing restricted type hierarchies (e.g., Result states).

42 Sealed class vs enum class?

Enum: fixed set of constants, each is single instance. Sealed: subclasses can have multiple instances, each can hold different state.

43 What is an enum class?

Represents a fixed set of constants: enum class Color { RED, GREEN } can have properties/methods.

44 What is an object declaration?

Singleton pattern using object keyword: object Singleton { fun doWork() }. Thread-safe lazy initialization.

45 What is a companion object?

Object inside a class marked companion. Members can be accessed like Java static: MyClass.factory().

46 Companion object vs Java static?

Companion objects are actual objects that can implement interfaces, extend classes, and be passed around. More flexible.

47 What is delegation in Kotlin?

Design pattern where object delegates responsibility to another. Kotlin has native support with by keyword.

48 What is class delegation?

Allows a class to delegate interface implementation to another object: class Derived(b: Base) : Base by b.

49 Inner vs nested classes?

Nested class is static by default (no outer reference). Inner class inner class Inner holds reference to outer class.

50 Difference inner class and nested class?

Inner class can access outer class members; nested class cannot without outer instance. Memory overhead for inner.

class Outer { inner class Inner { fun getOuter() = this@Outer } }

⚙️ 7. Advanced OOP Questions

51 Composition vs inheritance?

Composition: "has-a" relationship (e.g., Car has Engine). More flexible than inheritance ("is-a"). Prefer composition over inheritance.

52 What is dependency inversion principle (DIP)?

High-level modules should not depend on low-level modules; both depend on abstractions. Kotlin interfaces help.

53 How Kotlin supports SOLID?

Interfaces & abstract classes for OCP, LSP via proper inheritance, ISP via smaller interfaces, DIP via abstraction.

54 What is immutability and why important?

State cannot change after creation. Thread-safe, easier reasoning. Kotlin encourages val, data classes with copy.

55 How data classes support immutability?

Use val properties in primary constructor. copy() creates new instances with modified values.

56 What is copy function in data classes?

Generates a copy of data class instance optionally changing some fields: user.copy(name = "New").

57 What are equals(), hashCode(), toString()?

Universal methods from Any. equals for structural equality, hashCode for hash-based collections, toString for string representation.

58 How generated in data classes?

Compiler automatically generates meaningful equals/hashCode/toString based on primary constructor properties.

59 What is smart casting?

Kotlin automatically casts type after type-check: if (x is String) println(x.length). Works with OOP hierarchies.

60 OOP best practices in Kotlin?

Prefer immutability, composition over inheritance, use sealed classes for restricted hierarchies, data classes for simple data, visibility modifiers.

🏗️ 8. Scenario-Based / Practical Questions

61 Design Vehicle, extend Car and Bike.
open class Vehicle(val brand: String) { open fun move() = "moving" }
class Car(brand: String, val doors: Int) : Vehicle(brand) { override fun move() = "drive" }
class Bike(brand: String) : Vehicle(brand) { override fun move() = "pedal" }
62 Model a payment system using interfaces.
interface Payment { fun pay(amount: Double): Boolean }
class CreditCard : Payment { override fun pay(amount: Double) = true }
class PayPal : Payment { override fun pay(amount: Double) = true }
63 Restrict inheritance in Kotlin?

Mark class as final (default) or sealed for restricted subclasses within same file.

64 Create a singleton in Kotlin?

Use object Singleton { ... } or companion object + private constructor.

65 Hierarchy using sealed classes.
sealed class Result
data class Success(val data: String) : Result()
data class Error(val msg: String) : Result()
66 Refactor inheritance-heavy code using composition.

Instead of extending, pass dependencies: class Engine { fun start() } class Car(private val engine: Engine)

💻 9. Kotlin Code-Based Questions

67 Output of program with overridden functions?
open class A { open fun f() = "A" }
class B : A() { override fun f() = "B" }
fun main() { val a: A = B(); println(a.f()) } // prints "B"
68 Identify mistake in inheritance example.

Missing open on superclass or function. Example: class Base { fun test() } class Derived : Base() { override fun test() } → error: 'test' not open.

69 Convert Java OOP example into Kotlin.

Java: public class Dog { private String name; public Dog(String n){name=n;} } → Kotlin: class Dog(val name: String)

70 Lifecycle of an object in Kotlin.

Creation (constructor + init block) → usage → when no references, GC eligible. finalize() not recommended; use closeable or use blocks.
