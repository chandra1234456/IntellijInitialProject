Android Problem Solving Interview Questions

1. Basic Android Thinking Questions

  1. What happens internally when an Android app is launched from the launcher?
  2. Why must UI updates happen on the main thread? What problems occur if they run on a background thread?
  3. How would you prevent an app from crashing due to NullPointerException in production?
  4. What is the difference between compile-time error, runtime crash, and ANR?
  5. If an Activity is rotated, what exactly happens to the Activity instance?
  6. Why does Android destroy and recreate Activities during configuration changes?
  7. If your app crashes only on low RAM devices, how would you debug it?
  8. What is the difference between dp, sp, and px and when should each be used?
  9. How would you design an app to support different screen sizes efficiently?
  10. Why should long-running tasks not run inside onCreate()?

2. Android Lifecycle Problem Questions

  1. An Activity starts a network call, but the user rotates the screen. What problems might happen?
  2. How would you prevent duplicate API calls after screen rotation?
  3. A video is playing in an Activity and the screen rotates. How would you continue playback smoothly?
  4. What lifecycle method would you use to save UI state?
  5. How would you restore the scroll position of a RecyclerView after rotation?

3. Threading & Performance Problems

  1. Your app freezes for 5 seconds when opening a screen. How would you investigate it?
  2. What tools would you use to detect ANR issues?
  3. Suppose an image download takes 10 seconds. How would you ensure the UI remains responsive?
  4. How would you cancel a background task when the user leaves the screen?
  5. How do you avoid memory leaks caused by background threads?

4. Memory & Optimization Questions

  1. Why do Context leaks happen in Android?
  2. How can a static reference cause a memory leak?
  3. An app crashes with OutOfMemoryError while loading images. How would you fix it?
  4. How would you detect memory leaks in a production app?
  5. Why should you avoid loading large images directly into memory?

5. RecyclerView Problem Solving

  1. Why might a RecyclerView scroll lag or stutter?
  2. What happens if you don't implement ViewHolder pattern correctly?
  3. A RecyclerView shows wrong data when scrolling fast. Why?
  4. How would you implement pagination (infinite scrolling)?
  5. How do you efficiently update only changed items in RecyclerView?

6. Architecture Thinking Questions

  1. Why is putting all logic inside Activity considered bad practice?
  2. What problems does MVVM architecture solve?
  3. When should you use Repository pattern?
  4. Why are ViewModels lifecycle-aware?
  5. What happens to ViewModel when Activity is rotated?

7. Debugging Real Problems

  1. Your API works in Postman but fails in the Android app. What steps would you take?
  2. An app works fine in debug build but crashes in release build. Why could this happen?
  3. A feature works on Android 13 but crashes on Android 10. How would you debug it?
  4. How would you trace a crash reported in Play Store console?
  5. If users report random crashes but you cannot reproduce them, what would you do?

8. Advanced Problem Thinking

  1. How would you design an offline-first Android app?
  2. Suppose your app must load 1 million items efficiently, how would you design the UI?
  3. How would you reduce app startup time?
  4. What strategies would you use to reduce APK size?
  5. How would you secure sensitive user data in an Android app?
