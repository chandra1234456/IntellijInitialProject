4. Interview Questions (Important)

Q1: What is ViewModel?

It is a lifecycle-aware class used to store UI-related data and survive configuration changes.

Q2: Does ViewModel survive app kill?

No. It only survives configuration changes, not process death.

Q3: Difference between LiveData and StateFlow?

LiveData is lifecycle-aware, StateFlow is coroutine-based and more modern.

Q4: What is scope in ViewModel?

Scope is the lifecycle owner (Activity/Fragment) that controls ViewModel lifetime.

Q5: Why ViewModel is used?

  • To store UI data safely
  • To survive rotation
  • To separate UI and business logic

Q6: What happens on rotation?

Activity is recreated but ViewModel remains same instance.

5. Final Summary

  • MVVM separates UI, logic, and data
  • ViewModel survives configuration changes
  • LiveData = lifecycle aware
  • StateFlow = modern reactive state holder
  • Shared ViewModel enables fragment communication
