Android ViewModel

1. What is ViewModel?

ViewModel is a Jetpack Architecture Component designed to store and manage UI-related data in a lifecycle-conscious way.

2. What does “Scope” mean?

Scope = the lifecycle owner (Activity/Fragment) that holds the ViewModel

  • Activity scope → ViewModel belongs to Activity
  • Fragment scope → ViewModel belongs to Fragment

3. What does “Alive” mean?

👉 Alive = ViewModel is still stored in ViewModelStore and not cleared.
❌ Not alive = Activity is finished and ViewModel is destroyed.

4. ViewModel survives Configuration Change (Rotation)

✔ Activity is destroyed and recreated
✔ ViewModel remains SAME instance

5. ViewModel destroyed on Finish (Back Press)

❌ User presses back → Activity finishes → ViewModel destroyed

6. ViewModel Example

ViewModel

class UserViewModel : ViewModel() { private val _users = MutableLiveData<List<User>>() val users: LiveData<List<User>> = _users fun loadUsers() { _users.value = listOf(User("John"), User("Emma")) } }

Activity

class MainActivity : AppCompatActivity() { private val viewModel: UserViewModel by viewModels() override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) viewModel.users.observe(this) { // Update UI } } }

7. Key Summary

  • Scope = lifecycle owner (Activity/Fragment)
  • Alive = stored in ViewModelStore
  • Rotation → ViewModel survives
  • Finish → ViewModel destroyed
