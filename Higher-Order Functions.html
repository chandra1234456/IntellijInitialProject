🧠 Higher-Order Functions (HOFs) – Kotlin Interview Guide

✅ What is a Higher-Order Function?

A Higher-Order Function is a function that:

  • Takes another function as a parameter OR
  • Returns another function

🔹 Basic Example

fun operate(a: Int, b: Int, operation: (Int, Int) -> Int): Int { return operation(a, b) } val result = operate(5, 3) { x, y -> x + y } // 8

🔹 Function Types

(Int, Int) -> Int
  • Parameters → (Int, Int)
  • Return Type → Int

🔥 Common HOFs in Kotlin

map()

nums.map { it * 2 }

filter()

nums.filter { it > 1 }

forEach()

nums.forEach { println(it) }

reduce()

nums.reduce { acc, i -> acc + i }

fold()

nums.fold(10) { acc, i -> acc + i }

any / all / none

nums.any { it > 2 } nums.all { it > 0 } nums.none { it < 0 }

🔥 Lambda Expressions

val add = { a: Int, b: Int -> a + b } println(add(2, 3))

🔥 Trailing Lambda

operate(2, 3) { x, y -> x + y }

🚀 Returning a Function

fun multiplier(factor: Int): (Int) -> Int { return { number -> number * factor } } val double = multiplier(2) println(double(5))

🧩 Inline Functions

inline fun calculate(a: Int, b: Int, op: (Int, Int) -> Int): Int { return op(a, b) }

✔ Avoids lambda object creation
✔ Improves performance

⚡ noinline & crossinline

inline fun example( noinline normal: () -> Unit, crossinline cross: () -> Unit )
  • noinline → prevents inlining
  • crossinline → prevents non-local return

🔥 Interview Questions

Q1. What is a Higher-Order Function?

A function that takes or returns another function.

Q2. Benefits of HOF?

  • Code reusability
  • Cleaner code
  • Functional programming support

Q3. map vs forEach

map forEach
Returns new list Returns Unit
Transformation Side effects

Q4. Inline function?

Improves performance by avoiding lambda object creation.

Q5. Closure?

Lambda capturing outer variables.

var sum = 0 nums.forEach { sum += it }

Q6. reduce vs fold

  • reduce → no initial value
  • fold → has initial value

🎯 Coding Example

val result = listOf(1, 2, 3, 4) .filter { it % 2 == 0 } .map { it * it } .sum()

🏁 Key Takeaways

  • HOF = function using function
  • Lambda = short function syntax
  • Inline = performance optimization
  • map/filter = most used
