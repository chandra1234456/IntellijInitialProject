Android Design Patterns - Interview Prep

1. Builder Pattern ⭐

Q1: What is Builder Pattern?
Used to construct complex objects step by step instead of long constructors.
Q2: Where is it used in Android?
AlertDialog, Retrofit, Notification Builder.
Q3: Why use it?
Improves readability and avoids telescoping constructors.
AlertDialog.Builder(this) .setTitle("Hello") .setMessage("Welcome") .create()

2. Singleton Pattern ⭐

Q1: What is Singleton?
A class that has only one instance throughout the app.
Q2: Android usage?
Database, Retrofit, Repository.
Q3: Kotlin implementation?
object ApiClient { fun callApi() { println("API Called") } }

3. Observer Pattern ⭐

Q1: What is Observer Pattern?
One object notifies others when data changes.
Q2: Android example?
LiveData, Flow, StateFlow.
Q3: Why important?
Automatically updates UI when data changes.
viewModel.data.observe(this) { value -> textView.text = value }

4. Adapter Pattern ⭐

Q1: What is Adapter Pattern?
Converts one interface into another.
Q2: Android usage?
RecyclerView Adapter.
Q3: Why needed?
To display data in list format.
class MyAdapter : RecyclerView.Adapter()

5. Factory Pattern ⭐

Q1: What is Factory Pattern?
Creates objects without exposing creation logic.
Q2: Android usage?
ViewModelFactory.
class CarFactory { fun getCar(type: String): Car { return when(type) { "SUV" -> SUV() else -> Sedan() } } }

6. Dependency Injection (Hilt) ⭐

Q1: What is DI?
Objects are provided instead of being created inside class.
Q2: Android tool?
Hilt (recommended).
@HiltViewModel class MyViewModel @Inject constructor( private val repo: Repository )
