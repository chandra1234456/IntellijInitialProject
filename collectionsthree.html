Real-World Kotlin Collections

🔹 Scenario-Based Questions

1️⃣ You receive a list of users from API. You must show only active users in RecyclerView. How?
Use filter() to remove inactive users. 
val activeUsers = users.filter { it.isActive }
adapter.submitList(activeUsers)
Why? → filter() creates a clean immutable list for UI.
2️⃣ You must update RecyclerView efficiently when data changes. Which collection practice do you use?
Use immutable List with DiffUtil or ListAdapter. 
adapter.submitList(newList)
Avoid MutableList in adapters to prevent UI bugs.
3️⃣ API returns duplicate items. How do you remove duplicates?
Use distinct() or distinctBy(). 
val uniqueItems = items.distinctBy { it.id }
4️⃣ You need fast lookup of data by ID (caching). Which collection?
Use Map for O(1) lookup. 
val userCache = users.associateBy { it.id }
val user = userCache[userId]
5️⃣ Large dataset (10,000+ items) with multiple filters. How do you optimize?
Use Sequence for lazy evaluation. 
val result = items.asSequence()
    .filter { it.isActive }
    .map { it.name }
    .toList()
6️⃣ You must group chat messages by date. Which function?
Use groupBy(). 
val groupedMessages = messages.groupBy { it.date }
7️⃣ You need to sort products by price before displaying.
Use sortedBy() or sortedDescending(). 
val sortedProducts = products.sortedBy { it.price }
8️⃣ You only want to perform an action, not create a new list. What do you use?
Use forEach(). 
users.forEach { log(it.name) }
9️⃣ You must transform API model into UI model.
Use map(). 
val uiModels = apiModels.map {
    UserUI(it.id, it.name)
}
🔟 You want to expose data safely from ViewModel. Which collection?
Expose immutable List. 
private val _users = mutableListOf()
val users: List get() = _users
1️⃣1️⃣ API response can be null. How do you avoid crash?
Use safe calls and Elvis operator. 
val count = users?.size ?: 0
1️⃣2️⃣ You need the first matching item only. What do you use?
Use firstOrNull(). 
val admin = users.firstOrNull { it.role == "ADMIN" }
1️⃣3️⃣ When would you prefer Set over List in real apps?
When uniqueness matters (e.g. selected IDs). 
val selectedIds = mutableSetOf()
1️⃣4️⃣ How do you combine two lists from different APIs?
Use plus(). 
val combined = list1 + list2
1️⃣5️⃣ Why avoid MutableList in UI layers?
It can cause unexpected UI state changes and DiffUtil failures.
Best Practice → Mutable inside ViewModel, Immutable for UI

📌 Final Interview Takeaway

Real-world Kotlin collection usage focuses on:
• Immutability for UI • Performance using Sequence • Correct function selection (map, filter, groupBy) • Clean & safe data flow

🧠 Easy Memory Rule

Transform → map()
Select → filter()
Group → groupBy()
Lookup → Map
Large data → Sequence
