OOP (Object-Oriented Programming) Interview Guide

1. What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm based on objects, which contain data (variables) and behavior (methods).

2. Class vs Object

  • Class: Blueprint/template
  • Object: Instance of a class
class Car { String brand; int speed; void drive() { System.out.println("Car is driving"); } } public class Main { public static void main(String[] args) { Car c1 = new Car(); // Object c1.brand = "BMW"; c1.drive(); }

3. Four Pillars of OOP

Encapsulation

Wrapping data and methods into a single unit.

class Person { private String name; public String getName() { return name; } public void setName(String name) { this.name = name; } }

Abstraction

Hiding implementation details.

abstract class Animal { abstract void sound(); } class Dog extends Animal { void sound() { System.out.println("Bark"); } }

Inheritance

One class inherits another.

class Vehicle { void start() { System.out.println("Vehicle started"); } } class Bike extends Vehicle { }

Polymorphism

Same method, different behavior.

// Method Overloading class MathUtils { int add(int a, int b) { return a + b; } int add(int a, int b, int c) { return a + b + c; } }

  • What is encapsulation with real example?
  • Difference between abstraction and encapsulation?
  • What is method overloading vs overriding?
  • What is multiple inheritance?

5. Scenario-Based Questions

Scenario 1

You want to hide sensitive data like salary.

Answer: Use encapsulation with private variables.

Scenario 2

You have different payment methods (UPI, Card, Cash).

Answer: Use polymorphism (common interface).

Scenario 3

You want common functionality for all vehicles.

Answer: Use inheritance.

Scenario 4

You want to expose only essential features of a system.

Answer: Use abstraction.

6. Real-World Example

interface Payment { void pay(); } class UPI implements Payment { public void pay() { System.out.println("Paid via UPI"); } } class CreditCard implements Payment { public void pay() { System.out.println("Paid via Card"); } }

7. Common Mistakes

  • Confusing abstraction with encapsulation
  • Not using access modifiers properly
  • Overusing inheritance instead of composition

8. Tips

  • Use real-world examples in interviews
  • Explain with diagrams if possible
  • Focus on "why" not just "what"
