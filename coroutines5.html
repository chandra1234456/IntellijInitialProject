Android Coroutines Interview Questions & Answers

Beginner Level

1. Difference between Thread and Coroutine?
Threads are heavy-weight and managed by OS, while coroutines are lightweight and managed by Kotlin. Coroutines are faster, consume less memory, and avoid ANR when used properly.
2. Difference between launch and async?
launch does not return a result and returns a Job. async returns a Deferred object and is used when a result is needed.
3. Explain Dispatchers.
Main: UI operations
IO: Network, database, file operations
Default: CPU-intensive tasks like sorting and calculations
4. What is a suspend function?
A suspend function can suspend execution without blocking a thread and can only be called from another suspend function or coroutine.

Intermediate Level

5. Sequential vs Parallel API calls?
Sequential calls run one after another. Parallel calls use async/await to execute simultaneously.
6. Why is GlobalScope discouraged?
GlobalScope lives as long as the app, causing memory leaks. Use lifecycleScope or viewModelScope instead.
7. Exception handling in coroutines?
Use try-catch inside coroutine or CoroutineExceptionHandler for uncaught exceptions.
8. Job vs Deferred?
Job manages lifecycle. Deferred extends Job and returns a value using await().
9. How to cancel a coroutine?
Call job.cancel(). All child coroutines are cancelled automatically.

Advanced Level

10. What is structured concurrency?
It ensures coroutines are launched within a scope and are cancelled together, preventing leaks.
11. Job vs SupervisorJob?
Job cancels all children if one fails. SupervisorJob allows other children to continue.
12. withContext vs launch?
withContext switches thread and returns result. launch starts a new coroutine without blocking parent.
13. Timeout handling?
Use withTimeout or withTimeoutOrNull to cancel coroutine after a time limit.
14. What is coroutine leak?
When a coroutine outlives its lifecycle owner. Prevent using lifecycleScope and viewModelScope.

Android-Specific

15. API call in ViewModel?
Use viewModelScope.launch with Dispatchers.IO and expose data via LiveData or StateFlow.
16. Room with coroutines?
Room supports suspend functions and Flow. It automatically runs queries on background thread.
17. Retry API call?
Use repeat() or retry() operator in Flow to retry API calls.
18. Combine multiple APIs?
Use async-await inside coroutine scope to call APIs in parallel.
19. Flow vs suspend?
suspend returns a single value. Flow emits multiple values over time.

Senior / Tricky

20. Cold vs Hot Flow?
Cold Flow starts on collection. Hot Flow (StateFlow/SharedFlow) emits regardless of collectors.
21. CoroutineContext?
It defines coroutine behavior like Dispatcher, Job, and ExceptionHandler.
22. Can coroutines cause ANR?
Yes, if heavy work runs on Dispatchers.Main.
23. delay vs Thread.sleep?
delay is non-blocking. Thread.sleep blocks the thread.
24. Backpressure in Flow?
Flow handles backpressure automatically using suspend functions.
25. Testing coroutines?
Use runTest, TestDispatcher, and TestScope for coroutine testing.
