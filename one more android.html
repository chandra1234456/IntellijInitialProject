Android Fundamentals

1
What is Android?
+

Android is an open-source, Linux-based operating system developed by Google for mobile devices.

Key Points: Created in 2003, acquired by Google in 2005, first commercial version in 2008.

Architecture Layers:

  1. Linux Kernel - Hardware abstraction layer
  2. Hardware Abstraction Layer (HAL) - Standard interfaces
  3. Android Runtime (ART) - Executes Dalvik bytecode
  4. Native C/C++ Libraries - Core system components
  5. Java API Framework - App building blocks
  6. System Apps - Pre-installed applications
2
What is Android SDK?
+

Android SDK (Software Development Kit) is a collection of tools for building Android apps.

// SDK includes: - Android Studio IDE - Android Debug Bridge (ADB) - Android Emulator - SDK Tools & Platform Tools - API Libraries - Documentation

Key Components:

  • Build Tools - Compile resources and code
  • Platform Tools - ADB, fastboot
  • Emulator - Virtual Android device
  • Support Libraries - Backward compatibility
3
What is Android NDK?
+

Android NDK (Native Development Kit) allows you to implement parts of your app using native-code languages (C and C++).

When to use NDK:

  • CPU-intensive operations (games, physics simulations)
  • Reuse existing C/C++ libraries
  • Low-level hardware operations
  • Performance-critical code sections
// Example: Simple JNI function extern "C" JNIEXPORT jstring JNICALL Java_com_example_app_MainActivity_stringFromJNI( JNIEnv* env, jobject /* this */) { return env->NewStringUTF("Hello from C++"); }
Note: Most apps don't need NDK. Use it only when you need native performance or to reuse existing C/C++ code.
4
What is the difference between Dalvik and ART?
+
Feature Dalvik ART
Compilation Just-In-Time (JIT) Ahead-Of-Time (AOT)
Performance Slower (compiles at runtime) Faster (compiled during install)
Battery More consumption Optimized
Storage Less storage needed More storage (pre-compiled)

ART Advantages:

  • Better performance (apps launch faster)
  • Improved battery life
  • Enhanced garbage collection
  • Better debugging support

Android Components

5
What are the four main components of Android?
+

Android apps are built using four fundamental components:

  1. Activities - Single screen with UI
  2. Services - Background operations
  3. Broadcast Receivers - System-wide announcements
  4. Content Providers - Data sharing between apps
// Activity Example public class MainActivity extends AppCompatActivity { @Override protected void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); setContentView(R.layout.activity_main); } }
6
What is the Android Manifest file?
+

AndroidManifest.xml is the configuration file that describes essential information about your app to the Android system.

<manifest package="com.example.app"> <uses-permission android:name="android.permission.INTERNET"/> <application android:icon="@mipmap/ic_launcher" android:label="@string/app_name"> <activity android:name=".MainActivity"> <intent-filter> <action android:name="android.intent.action.MAIN"/> <category android:name="android.intent.category.LAUNCHER"/> </intent-filter> </activity> </application> </manifest>

Contains:

  • Package name and version
  • App components (activities, services, etc.)
  • Permissions required
  • Minimum and target SDK versions
  • Hardware/software features required
7
What are Intents in Android?
+

Intents are messaging objects used to request an action from another app component.

Two types of Intents:

1. Explicit Intents - Specify the exact component to start

// Explicit Intent Example Intent intent = new Intent(this, SecondActivity.class); startActivity(intent);

2. Implicit Intents - Declare a general action to perform

// Implicit Intent Example Intent intent = new Intent(Intent.ACTION_VIEW); intent.setData(Uri.parse("https://www.example.com")); startActivity(intent);

Common Intent Actions:

  • ACTION_VIEW - Display data to user
  • ACTION_SEND - Send data to another app
  • ACTION_DIAL - Open dialer with number
  • ACTION_CALL - Initiate phone call
8
What is the difference between Service and IntentService?
+
Aspect Service IntentService
Thread Runs on main thread Worker thread
Multiple Requests Handle manually Queued automatically
Stopping Manual stop required Stops automatically
Use Case Music player, download manager Short background tasks
Note: IntentService is deprecated in API 30. Use WorkManager or JobIntentService instead.

Advanced Android Topics

9
What is ViewModel and LiveData?
+

ViewModel - Stores and manages UI-related data in a lifecycle conscious way.

LiveData - Observable data holder class that respects lifecycle.

// ViewModel with LiveData Example public class UserViewModel extends ViewModel { private MutableLiveData userName = new MutableLiveData<>(); public LiveData getUserName() { return userName; } public void setUserName(String name) { userName.setValue(name); } } // In Activity UserViewModel model = ViewModelProviders.of(this).get(UserViewModel.class); model.getUserName().observe(this, name -> { // Update UI with new name textView.setText(name); });

Benefits:

  • Survives configuration changes (rotation)
  • No memory leaks (auto cleanup)
  • Lifecycle awareness
  • Data persistence
10
What is Dependency Injection in Android?
+

Dependency Injection (DI) is a technique where objects receive their dependencies from external sources rather than creating them internally.

Common DI frameworks:

  • Dagger 2 - Most popular, compile-time DI
  • Hilt - Built on Dagger, simplified for Android
  • Koin - Kotlin-friendly, lightweight
// Without DI class Car { private Engine engine = new Engine(); // Direct dependency } // With DI class Car { private Engine engine; // Dependency injected via constructor public Car(Engine engine) { this.engine = engine; } }

Benefits:

  • Better code reusability
  • Easier testing (mock dependencies)
  • Loose coupling
  • Centralized configuration
11
What is Jetpack Compose?
+

Jetpack Compose is Android's modern toolkit for building native UI declaratively.

// Compose UI Example @Composable fun Greeting(name: String) { Column( modifier = Modifier .fillMaxSize() .padding(16.dp) ) { Text( text = "Hello, $name!", fontSize = 24.sp, color = MaterialTheme.colorScheme.primary ) Button(onClick = { /* Handle click */ }) { Text("Click me") } } }

Advantages over XML:

  • Declarative UI
  • Less code
  • Live preview
  • Kotlin-based
  • State management built-in
Note: Compose is the future of Android UI development. New projects should consider using it.
12
What is the difference between API levels?
+

Android API levels represent different versions of Android with varying features and capabilities.

Key API Levels:

  • API 21 (Lollipop) - Material Design, ART runtime
  • API 23 (Marshmallow) - Runtime permissions
  • API 26 (Oreo) - Notification channels, background limits
  • API 28 (Pie) - Biometric authentication
  • API 29 (Android 10) - Scoped storage, dark theme
  • API 30 (Android 11) - One-time permissions
  • API 31+ (Android 12+) - Material You, privacy dashboard
Best Practice: Target latest API, support older APIs using compatibility libraries. Check current distribution on official Android dashboard.
