Advanced Java Collections – Scenario Based Problems

1. LRU Cache Design Design a cache with fixed capacity that removes the least recently used item when full. Explain which Collection types should be used and why (ordering + O(1) access required).
Expected Behavior: Oldest unused entry removed automatically.
2. Detect First Non-Repeated Character Given a stream of characters, continuously return the first non-repeating character. Must work efficiently for large streams.
Input: aabbcdde → Output: c
3. Group Anagrams Given a list of words, group words that are anagrams of each other. Explain the internal key structure used.
Input: [eat, tea, tan, ate, nat] → Output: [[eat, tea, ate], [tan, nat]]
4. Top K Frequent Elements Given an array of integers, return the top K frequent elements. Explain why a PriorityQueue or Heap is efficient.
Input: [1,1,1,2,2,3], K=2 → Output: [1,2]
5. Thread-Safe Counter System Multiple threads update a shared Map of product counts. How would you prevent data inconsistency?
Expected: Accurate counts without race conditions.
6. Remove Duplicates but Maintain Order Given a list with duplicates, remove duplicates while preserving insertion order. Explain correct Set implementation.
Input: [3,1,2,3,1] → Output: [3,1,2]
7. Real-Time Leaderboard Maintain players sorted by score with frequent updates. Explain best Collection choice for fast reordering.
Expected: Always sorted ranking.
8. Merge Overlapping Intervals Store and merge time intervals efficiently. Explain sorting and merging strategy.
Input: [1,3],[2,6] → Output: [1,6]
9. Find Pair with Given Sum Given a large dataset, find if any two numbers sum to target. Explain why Set improves performance.
Input: [2,7,11,15], target=9 → Output: true
10. Design Undo/Redo System Implement undo and redo functionality. Explain how Deque or Stack helps.
Expected: Reversible operations.
