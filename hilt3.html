Dependency Injection (DI)

What is Dependency?

A dependency occurs when one class needs another class to function.

A class depends on another class when it needs it to work.

Real Understanding

The earlier idea that "student depends on college" is misleading. A student can exist independently. So dependency means usage, not existence.

Without Dependency Injection (Tightly Coupled)

```

class Engine {
void start() {
System.out.println("Engine started");
}
}

class Car {
Engine engine = new Engine(); // Car creates Engine

```
void drive() {
    engine.start();
    System.out.println("Car is driving");
}
```

}
```

❌ Problems:

  • Car is tightly coupled with Engine
  • Hard to test
  • Cannot easily replace Engine

With Dependency Injection (Loosely Coupled)

```

class Engine {
void start() {
System.out.println("Engine started");
}
}

class Car {
Engine engine;

```
Car(Engine engine) {
    this.engine = engine; // Injected from outside
}

void drive() {
    engine.start();
    System.out.println("Car is driving");
}
```

}
```

✅ Benefits:

  • Car does NOT create Engine
  • Easy to replace dependencies
  • Better testing
  • Flexible and reusable code

Android Example

Without DI

```

class MainActivity extends AppCompatActivity {
Database db = new Database(); // tightly coupled
}
```

With DI

```

class MainActivity extends AppCompatActivity {
Database db;

```
public MainActivity(Database db) {
    this.db = db;
}
```

}
```

In real Android apps, tools like Hilt and Dagger provide dependencies automatically.

Types of Dependency Injection

  • Constructor Injection (Recommended)
  • Setter Injection
  • Field Injection

Easy Definition

Dependency Injection = "Don’t create objects yourself, let someone else provide them."
