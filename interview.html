Android Interview Questions (3–6 Years Experience)

1. What is MVVM?

MVVM (Model-View-ViewModel) separates concerns: Model → Data layer View → UI ViewModel → Handles logic & provides data to UI using LiveData/StateFlow

2. MVVM vs MVP

MVVM → Uses LiveData, no direct View reference MVP → Presenter directly updates View using interfaces MVVM is more testable and less boilerplate

3. Repository Role

Acts as single source of truth and manages API + DB.

4. Data in Orientation Change

Use ViewModel or SavedStateHandle.

5. Dependency Injection

Objects get dependencies externally (Hilt/Dagger). Improves testability and reduces coupling.

6. Clean Architecture

Layers: Presentation → UI Domain → Business logic Data → API/DB Improves maintainability.

7. LiveData

Lifecycle-aware observable data holder.

8. ViewModel

Stores UI data and survives configuration changes.

9. SQLite

Lightweight local database using SQL.

10. SQLite vs Room

SQLite → Manual queries Room → Abstraction with annotations & DAO

11. Secure Room DB

Use SQLCipher and Android Keystore.

12. SharedPreferences

Pros → Simple Cons → Not secure, limited storage

13. Encrypted SharedPreferences

Uses AES encryption + Keystore.

14. Kotlin Features

Null safety, coroutines, concise syntax.

15. lazy vs lateinit

lazy → val, initialized on first use lateinit → var, must initialize before use

16. Extension Functions

fun String.isPalindrome(): Boolean { return this == this.reversed() }

17. Higher-order Functions

fun operate(a: Int, b: Int, op: (Int, Int) -> Int): Int { return op(a, b) }

18. Sealed Classes

sealed class Result { data class Success(val data: String): Result() data class Error(val msg: String): Result() }

19. Coroutines

Lightweight threads for async tasks using suspend functions.

20. Threads

Main → UI IO → Network/DB Default → CPU tasks

21. Dispatchers

Main → UI IO → network/db Default → CPU tasks

22. WorkManager vs Executors

WorkManager → persistent tasks Executors → temporary tasks

23. launch vs async

launch → no result async → returns result

24. App Security

Use HTTPS, encryption, ProGuard, biometrics

25. ProGuard

Obfuscates code and reduces APK size.

26. Android Keystore

Secure storage for cryptographic keys.

28. Biometrics

Use BiometricPrompt API.

29. Activity Lifecycle

onCreate → init onStart → visible onResume → interactive

30. Activity vs Fragment

Activity → full screen Fragment → reusable UI component

31. Intents

Explicit → specific component Implicit → any matching app

32. Jetpack

Set of libraries like Room, Navigation, LiveData

33. Jetpack Compose

@Composable fun Greeting(name: String) { Text("Hello $name") }

34. Content Provider

Used for sharing data between apps.

35. Offline Caching

Room + Network sync + WorkManager

36. DataStore

Better than SharedPreferences, async & safe.

37. Null Safety

38. Inline Functions

Improve performance by avoiding object creation.

39. Suspend Functions

suspend fun fetchData() { }

40. Hilt DI
