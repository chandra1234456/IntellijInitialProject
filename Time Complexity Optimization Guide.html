How to Identify & Reduce Time Complexity

Step 1: How to Identify Time Complexity

1️⃣ Count the Loops

  • No loop → O(1)
  • One loop → O(n)
  • Two nested loops → O(n²)
  • Three nested loops → O(n³)

2️⃣ Check if Loop Depends on Input Size

for (i in 0 until arr.size)

If arr.size = n → Loop runs n times → O(n)

3️⃣ Check for Recursion

Each recursive call adds to complexity.

4️⃣ Check Expensive Operations Inside Loop

if (list.contains(x))

If contains() is O(n) and used inside loop → O(n²)

Step 2: How to Reduce Time Complexity

Remove Nested Loops

for (i in arr.indices) {
    for (j in arr.indices) {
        if (arr[i] == arr[j]) {
            println("Duplicate")
        }
    }
}

Time = O(n²)

Optimized Using Set

val set = mutableSetOf()

for (num in arr) {
    if (set.contains(num)) {
        println("Duplicate")
    }
    set.add(num)
}

Time = O(n)

Replace nested loops with HashSet or HashMap when possible.

Business Example Optimization

Check if Two Arrays Have Common Element

Brute Force (O(n²))

for (i in arr1) {
    for (j in arr2) {
        if (i == j) return true
    }
}

Optimized (O(n))

val set = arr1.toHashSet()

for (num in arr2) {
    if (set.contains(num)) return true
}

How Interviewers Think

  • Can you identify complexity?
  • Can you remove nested loops?
  • Can you use better data structure?
  • Can you trade space for time?

Golden Optimization Rules

  • Replace nested loops with HashMap / HashSet
  • Sort once, then use Binary Search
  • Precompute values outside loop
  • Avoid repeated calculations
  • Choose correct data structure

Important Self-Check Questions (With Answers)

1️⃣ What is n?

Answer: n represents the input size.
Example: If array has 100 elements → n = 100
Always identify what grows when input grows.

2️⃣ How many times does this run?

Answer: Count how many times the loop executes.
If loop runs from 0 to n → O(n)
If nested loop runs n × n → O(n²)

3️⃣ Is there loop inside loop?

Answer: Yes → usually O(n²)
Example: 

for (i in arr) {
    for (j in arr) {
        println(i + j)
    }
}

4️⃣ Is there searching inside loop?

Answer: If using contains() inside loop, and contains() takes O(n), then total becomes O(n²).
Replace list with HashSet to reduce lookup to O(1).

5️⃣ Can I reduce lookup time to O(1)?

Answer: Use:

  • HashSet for fast search
  • HashMap for key-value lookup
  • Array index access
These provide O(1) average lookup time.

Always think: Can I replace searching with hashing?
