Android MVVM + Clean Architecture + Hilt

1. Project Structure

com.example.app │ ├── presentation/ │ ├── ui/ │ │ ├── MainActivity.kt │ ├── viewmodel/ │ │ ├── UserViewModel.kt │ ├── domain/ │ ├── model/ │ │ ├── User.kt │ ├── repository/ │ │ ├── UserRepository.kt │ ├── usecase/ │ │ ├── GetUsersUseCase.kt │ ├── data/ │ ├── repository/ │ │ ├── UserRepositoryImpl.kt │ ├── remote/ │ │ ├── ApiService.kt │ ├── di/ │ ├── AppModule.kt │ └── MyApplication.kt

2. Clean Architecture Flow

UI → ViewModel → UseCase → Repository → API/DB

3. Domain Layer

Repository Interface

interface UserRepository { suspend fun getUsers(): List<User> }

UseCase

class GetUsersUseCase @Inject constructor( private val repository: UserRepository ) { suspend operator fun invoke(): List<User> { return repository.getUsers() } }

4. Data Layer

Repository Implementation

class UserRepositoryImpl @Inject constructor( private val api: ApiService ) : UserRepository { override suspend fun getUsers(): List<User> { return api.getUsers() } }

5. Presentation Layer (MVVM)

ViewModel

@HiltViewModel class UserViewModel @Inject constructor( private val getUsersUseCase: GetUsersUseCase ) : ViewModel() { val users = liveData { emit(getUsersUseCase()) } }

Activity

@AndroidEntryPoint class MainActivity : AppCompatActivity() { private val viewModel: UserViewModel by viewModels() override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) viewModel.users.observe(this) { // update UI } } }

6. Hilt Integration

Application Class

@HiltAndroidApp class MyApplication : Application()

Hilt Module

@Module @InstallIn(SingletonComponent::class) object AppModule { @Provides fun provideApi(): ApiService { return Retrofit.Builder() .baseUrl("https://api.example.com") .build() .create(ApiService::class.java) } @Provides fun provideRepository( api: ApiService ): UserRepository { return UserRepositoryImpl(api) } }

7. Scenario-Based Interview Question

Problem:

Build a feature to fetch a list of users from API and display in UI using MVVM + Clean Architecture + Hilt.

Expected Approach:

  • Create UseCase in domain layer
  • Create repository interface
  • Implement repository in data layer
  • Inject using Hilt
  • Call UseCase in ViewModel
  • Observe data in Activity/Fragment

8. Real Interview Coding Question

Question:

Design a login feature using MVVM + Clean Architecture.

Expected Layers:

Presentation → LoginViewModel Domain → LoginUseCase Data → AuthRepositoryImpl

Sample Code:

class LoginUseCase @Inject constructor( private val repo: AuthRepository ) { suspend operator fun invoke( username: String, password: String ): Boolean { return repo.login(username, password) } }

9. Common Mistakes

  • Putting business logic in ViewModel
  • Skipping domain layer
  • Not using interfaces
  • Wrong Hilt scope usage

10. Pro Tips

  • Always keep domain layer pure (no Android imports)
  • Use interfaces for flexibility
  • Keep ViewModel lightweight
