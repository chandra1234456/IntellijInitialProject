Dependency Injection (DI) with Hilt – Interview Guide

1. What is Dependency Injection?

Dependency Injection (DI) is a design pattern where dependencies are provided to a class instead of the class creating them itself.

2. What is Hilt?

Hilt is a dependency injection library for Android built on top of Dagger. It reduces boilerplate and simplifies DI setup.

3. Why use Hilt?

  • Reduces boilerplate code
  • Lifecycle-aware components
  • Easy integration with Android components
  • Improves testability

4. Basic Example

Step 1: Add Annotation

@HiltAndroidApp class MyApplication : Application()

Step 2: Inject Dependency

class Engine @Inject constructor() class Car @Inject constructor(private val engine: Engine)

Step 3: Use in Activity

@AndroidEntryPoint class MainActivity : AppCompatActivity() { @Inject lateinit var car: Car override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) car // ready to use } }

5. @Module and @Provides Example

@Module @InstallIn(SingletonComponent::class) object AppModule { @Provides fun provideEngine(): Engine { return Engine() } }

6. Scope Annotations

  • @Singleton

8. Scenario-Based Questions

Scenario 1:

You have multiple implementations of a repository. How will you inject a specific one?

Answer: Use @Qualifier annotation.

Scenario 2:

You want a single instance of a class across the app.

Answer: Use @Singleton scope.

Scenario 3:

How to inject dependencies into ViewModel?

@HiltViewModel class MyViewModel @Inject constructor( private val repository: Repo ) : ViewModel()

Scenario 4:

How to pass runtime parameters?

Answer: Use Assisted Injection.

9. Common Mistakes

  • Forgetting @AndroidEntryPoint
  • Wrong scope usage
  • Missing @HiltAndroidApp
  • Injecting before initialization
