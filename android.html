Android Basics Interview Questions

Essential questions for Android developer interviews with detailed answers

1. What is Android?

Android is an open-source, Linux-based operating system designed primarily for touchscreen mobile devices such as smartphones and tablets.

Key Characteristics:

  • Open Source: Android's source code is released under open-source licenses, allowing developers to modify and distribute the software
  • Linux-based Kernel: Provides core system services like security, memory management, process management, and device drivers
  • Developed by Google: Originally developed by Android Inc., which Google bought in 2005
  • App Ecosystem: Applications are primarily written in Java/Kotlin and distributed through Google Play Store
  • Multi-touch Support: Native support for multi-touch gestures
  • Multitasking: Supports true multitasking of applications
2. What is Android SDK?

The Android SDK (Software Development Kit) is a collection of tools, libraries, and documentation that developers use to create applications for the Android platform.

Main Components of Android SDK:

  • Android Studio: The official Integrated Development Environment (IDE) for Android development
  • Android Debug Bridge (ADB): A command-line tool for communicating with an Android device
  • Android Emulator: A virtual mobile device that runs on your computer to test applications
  • SDK Tools: Build tools, profilers, and other utilities
  • SDK Platform Tools: Platform-dependent tools for developing and debugging
  • Android Platform: A version of the Android operating system
  • System Images: Needed for the emulator to run different versions of Android
  • Support Libraries: Backward-compatible versions of Android framework APIs

Developers use the SDK to write apps in Java, Kotlin, or C++ and test them on emulators or real devices.

3. What is Android NDK?

The Android NDK (Native Development Kit) is a toolset that allows developers to implement parts of their Android apps using native-code languages like C and C++.

When to Use NDK:

  • Performance-Critical Code: For CPU-intensive operations like physics simulations, signal processing
  • Reuse Existing Libraries: To reuse C/C++ libraries in Android applications
  • Low-Level Operations: For operations that require direct hardware access
  • Game Development: For game engines and graphics-intensive applications

NDK vs SDK:

  • SDK: Uses Java/Kotlin, easier to develop, managed memory, platform-independent bytecode
  • NDK: Uses C/C++, more complex, manual memory management, platform-dependent native code

Most Android apps don't need the NDK. The SDK is sufficient for typical applications. NDK is used only when necessary for performance reasons or to reuse existing native code.

4. What is the Android Runtime (ART)?

ART (Android Runtime) is the managed runtime used by applications and some system services on Android. It replaced Dalvik (the original runtime) starting from Android 5.0 (Lollipop).

Key Features of ART:

  • Ahead-of-Time (AOT) Compilation: Compiles apps during installation rather than at runtime
  • Improved Performance: Better execution performance and battery optimization
  • Garbage Collection Improvements: More efficient garbage collection with less pause time
  • Development and Debugging Features: Support for sampling profiler, detailed diagnostic exceptions, and crash reporting

ART vs Dalvik:

  • Dalvik: Used Just-in-Time (JIT) compilation, compiled bytecode at runtime
  • ART: Uses Ahead-of-Time (AOT) compilation, compiles to native machine code during app installation
5. What are the key components of an Android application?

Android applications are composed of several essential components that define the app's structure and functionality:

Main Components:

  • Activities: Represent a single screen with a user interface (e.g., login screen, home screen)
  • Services: Run in the background to perform long-running operations or work for remote processes
  • Broadcast Receivers: Respond to system-wide broadcast announcements (e.g., low battery, screen off)
  • Content Providers: Manage a shared set of app data that can be stored in the file system, SQLite database, or on the web
  • Intents: Messaging objects used to request an action from another app component
  • Fragments: Represent a portion of user interface in an Activity, allowing modular design
  • Manifest File: Describes essential information about the app to the Android system
6. What is the Android Manifest file?

The AndroidManifest.xml file is a crucial file in every Android app that provides essential information about the application to the Android system.

Information Contained in Manifest:

  • Package Name: Unique application identifier
  • App Components: Declares all Activities, Services, Broadcast Receivers, and Content Providers
  • Permissions: Lists permissions the app needs to function (e.g., camera access, internet)
  • Minimum API Level: The minimum Android API level required
  • Target API Level: The API level the app is designed for
  • Hardware/Software Features: Declares any hardware or software features the app requires
  • App Metadata: Icon, label, theme, and other metadata

The Android system must see information in the manifest before it can run any of the app's code.
