Hilt Interview Questions - Android

Basic Hilt / DI Questions

1. What is Hilt?
Hilt is a dependency injection (DI) library built on top of Dagger that simplifies DI in Android apps.
2. Why use Hilt?
- Reduces boilerplate code
- Easier setup than Dagger
- Manages object creation automatically
- Handles Android lifecycle safely
3. Difference between Hilt and Dagger?
- Hilt is a wrapper over Dagger
- Hilt provides predefined components for Android
- Dagger requires manual component setup
4. What problem does Dependency Injection solve?
- Removes tight coupling
- Improves testability
- Makes code easier to maintain

Hilt Annotations

5. What is @HiltAndroidApp?
- Applied to Application class
- Triggers Hilt code generation
- Creates the Application-level component
@HiltAndroidApp
class MyApp : Application()
6. What is @AndroidEntryPoint?
- Used on Android classes (Activity, Fragment, Service)
- Allows Hilt to inject dependencies
7. What is @Inject?
- Used to tell Hilt how to create an object
- Can be used on constructors, fields, or methods
8. What is @Module?
- Used to provide dependencies that cannot be constructor-injected (e.g., Retrofit, Room)
9. Difference between @Provides and @Binds?
@Provides@Binds
Uses method bodyAbstract method
More flexibleFaster, less boilerplate

10. What are Hilt components?
They define the lifetime of dependencies.
ComponentLifetime

11. What is @Singleton?
Creates one instance for the entire app. Lives as long as the application.
12. How do you inject ViewModel using Hilt? 
@HiltViewModel
class MainViewModel @Inject constructor(
    private val repo: MainRepository
) : ViewModel()

Advanced / Scenario-Based Questions

13. How to inject Retrofit using Hilt? 
@Module
@InstallIn(SingletonComponent::class)
object NetworkModule {

    @Provides
    @Singleton
    fun provideRetrofit(): Retrofit {
        return Retrofit.Builder()
            .baseUrl("https://api.example.com/")
            .build()
    }
}
14. How to inject an interface?
Use @Binds
@Binds
abstract fun bindRepo(
    repoImpl: RepoImpl
): Repo
15. What is Qualifier in Hilt?
Used when multiple implementations of the same type exist.
@Qualifier
@Retention(AnnotationRetention.BINARY)
annotation class IoDispatcher
16. How do you test with Hilt?
- Use @HiltAndroidTest
- Replace modules using @TestInstallIn
