Git Interview Questions

üìö Git Fundamentals

1. What is Git?
Git is a distributed version control system that tracks changes in source code during software development. Created by Linus Torvalds in 2005 for Linux kernel development.
2. Difference between Git and GitHub?
Git: Version control system (software you install locally).
GitHub: Cloud-based hosting service for Git repositories (platform).
3. What are the three main areas in Git?
1. Working Directory - Where you modify files
2. Staging Area (Index) - Where you prepare commits
3. Repository (Git Directory) - Where Git stores metadata and commits
4. Explain the basic Git workflow
1. Modify files in working directory
2. Stage changes: git add
3. Commit changes: git commit
4. Push to remote: git push

üîÑ Git Commands & Operations

5. How to initialize a Git repository?
git init Creates a new .git directory in your project.
6. Difference between git fetch and git pull?
git fetch: Downloads changes from remote but doesn't merge
git pull: Downloads changes and merges into current branch (fetch + merge) git fetch origin git pull origin master
7. What is git clone?
Creates a copy of an existing repository. git clone https://github.com/user/repo.git
8. How to check Git status?
git status Shows: modified files, staged files, untracked files.
9. How to view commit history?
git log git log --oneline git log --graph --all

üåø Branching & Merging

10. How to create and switch branches?
git branch feature-branch # Create branch git checkout feature-branch # Switch to branch # OR combine: git checkout -b feature-branch
11. Difference between git merge and git rebase?
Merge: Creates a new commit that combines two branches
Rebase: Moves branch to tip of another branch (linear history) git merge feature-branch git rebase master
12. What is a merge conflict?
When Git cannot automatically resolve differences between two commits. You must manually resolve conflicts in affected files.
13. How to resolve merge conflicts?
1. Open conflicted files
2. Look for <<<<<<< markers
3. Resolve conflicts manually
4. Stage resolved files: git add
5. Commit: git commit

üîß Advanced Git

14. What is git stash?
Temporarily saves uncommitted changes. git stash # Save changes git stash list # View stashes git stash pop # Apply and remove stash git stash apply # Apply but keep stash
15. How to undo commits?
git reset --soft HEAD~1 # Undo commit, keep changes staged git reset --mixed HEAD~1 # Undo commit, keep changes unstaged git reset --hard HEAD~1 # Undo commit, discard changes git revert <commit-hash> # Create new commit that undoes changes
16. What is .gitignore file?
Specifies files/folders Git should ignore. # .gitignore example node_modules/ *.log .env .DS_Store
17. What is git cherry-pick?
Applies a specific commit from one branch to another. git cherry-pick <commit-hash>
18. Difference between HEAD, master, and origin?
HEAD: Pointer to current commit/branch
master/main: Default branch name
origin: Default remote repository name

üöÄ Git Workflows & Best Practices

19. What is Git Flow?
Popular branching model with:
  • master - Production code
  • develop - Integration branch
  • feature/* - New features
  • release/* - Release preparation
  • hotfix/* - Quick production fixes
20. What is a pull request (PR)?
Request to merge changes from one branch to another. Enables code review before merging.
21. What is squashing commits?
Combining multiple commits into one. git rebase -i HEAD~3 # Interactive rebase last 3 commits
22. How to check differences between files?
git diff # Unstaged changes git diff --staged # Staged changes git diff branch1..branch2 # Between branches git diff HEAD~3..HEAD # Last 3 commits
23. How to rename a branch?
git branch -m old-name new-name # Local git push origin --delete old-name # Delete old remote git push origin -u new-name # Push new branch
24. What is git tag?
Marks specific points in history (usually releases). git tag v1.0.0 # Lightweight tag git tag -a v1.0.0 -m "Release" # Annotated tag git push origin v1.0.0 # Push tag to remote

üîç Common Interview Scenarios

25. You committed to wrong branch. What to do?
# On wrong branch git log --oneline # Get commit hash git reset HEAD~1 # Remove commit from wrong branch git stash # Save changes git checkout correct-branch git stash pop git add . git commit -m "Message"
26. How to find which commit introduced a bug?
Use git bisect: git bisect start git bisect bad # Current commit is bad git bisect good v1.0 # v1.0 was good # Git checks out middle commit, you test it git bisect good/bad # Mark as good or bad # Repeat until Git finds the bad commit git bisect reset # End bisect session
27. How to recover deleted branch?
# Find commit hash git reflog # Restore branch git checkout -b branch-name <commit-hash>
28. How to update forked repository?
# Add upstream remote git remote add upstream https://github.com/original/repo.git # Fetch upstream changes git fetch upstream # Merge into your branch git checkout master git merge upstream/master # Push to your fork git push origin master

üí° Git Best Practices

  • Write meaningful commit messages
  • Commit small, logical changes
  • Pull before you push
  • Use .gitignore properly
  • Review changes before committing (git diff)
  • Create branches for new features
