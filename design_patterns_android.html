Android Design Patterns (Interview Guide)

1. Builder Pattern ⭐

Definition: Used to build complex objects step by step instead of using long constructors.

Android Example: AlertDialog, Retrofit

AlertDialog.Builder(this) .setTitle("Delete") .setMessage("Are you sure?") .setPositiveButton("Yes", null) .create()

Custom Example:

class User private constructor( val name: String, val age: Int ) { class Builder { private var name = "" private var age = 0 fun setName(name: String) = apply { this.name = name } fun setAge(age: Int) = apply { this.age = age } fun build() = User(name, age) } } // Usage: val user = User.Builder() .setName("John") .setAge(25) .build()

2. Singleton Pattern ⭐

Definition: Ensures only one instance of a class exists in the app.

Android Example: Database, Retrofit instance

object Database { val name = "AppDB" }

Custom Example:

object ApiClient { fun getData() { println("Fetching data...") } } // Usage: ApiClient.getData()

3. Observer Pattern ⭐

Definition: One object notifies others when data changes.

Android Example: LiveData, StateFlow

viewModel.text.observe(this) { value -> textView.text = value }

Custom Example:

interface Observer { fun update(data: String) } class DataSource { private val observers = mutableListOf() fun addObserver(observer: Observer) { observers.add(observer) } fun notifyData(data: String) { observers.forEach { it.update(data) } } }

4. Adapter Pattern ⭐

Definition: Converts one interface into another (used for lists).

Android Example: RecyclerView.Adapter

class MyAdapter : RecyclerView.Adapter()

Custom Example:

class MediaAdapter { fun playVideo(video: String) { println("Playing video: $video") } } class AudioPlayer { fun playAudio(audio: String) { println("Playing audio: $audio") } } // Adapter class PlayerAdapter(private val audio: AudioPlayer) { fun play(video: String) { audio.playAudio(video) } }

5. Factory Pattern ⭐

Definition: Creates objects without exposing creation logic.

Android Example: ViewModelFactory

class ViewModelFactory : ViewModelProvider.Factory { override fun create(modelClass: Class): T { return MyViewModel() as T } }

Custom Example:

class CarFactory { fun getCar(type: String): Car { return when(type) { "SUV" -> SUV() "Sedan" -> Sedan() else -> Sedan() } } }

6. Dependency Injection (Hilt) ⭐

Definition: Instead of creating objects, they are provided externally.

Android Example (Hilt):

@HiltViewModel class MyViewModel @Inject constructor( private val repo: Repository )

Custom Simple Example:

class Engine class Car(private val engine: Engine) fun main() { val engine = Engine() val car = Car(engine) }
