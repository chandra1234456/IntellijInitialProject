Android Interview Preparation

🧠 ViewModel

Basic

Q1. What is a ViewModel?
A lifecycle-aware component that stores and manages UI-related data.

Q2. Why use ViewModel?

  • Avoid data loss on rotation
  • Separate UI & business logic
  • Improve testability

Q3. Does it survive process death?
No, only configuration changes. Use SavedStateHandle.

Intermediate

Difference: ViewModel vs AndroidViewModel

  • ViewModel → no context
  • AndroidViewModel → has Application context

Advanced

Can it leak memory? Yes, if holding Activity reference.

Clearing: onCleared()

📡 LiveData

What is LiveData?
Lifecycle-aware observable data holder.

setValue vs postValue

  • setValue → main thread
  • postValue → background thread

Problems

  • No backpressure
  • Limited operators

🌊 Kotlin Flow

What is Flow?
Cold asynchronous stream.

Cold Stream? Starts only when collected.

Feature Flow LiveData
Lifecycle aware No Yes
Type Cold Hot

🔥 StateFlow

What is StateFlow?
Hot flow holding current state.

Key Points

  • Always has value
  • Best for UI state
  • Thread-safe

⚡ SharedFlow

What is SharedFlow?
Hot flow for events.

Feature StateFlow SharedFlow
State Yes No
Use Case UI State Events

💡 Scenarios

StateFlow vs SharedFlow

  • StateFlow → UI state
  • SharedFlow → events

Convert Flow to LiveData

flow.asLiveData()

Collect Flow safely

lifecycleScope.launch { repeatOnLifecycle(Lifecycle.State.STARTED) { flow.collect { } } }

🎯 Coding

StateFlow Example

private val _state = MutableStateFlow(0) val state: StateFlow<Int> = _state

SharedFlow Example

private val _events = MutableSharedFlow<String>() val events = _events.asSharedFlow()

🧩 Key Takeaways

  • StateFlow = State
  • SharedFlow = Events
  • Flow = Streams
